Order Pool Monitoring Dashboard

Business Context

In many companies, customer orders can be placed in an operational “pool” when they require validation before being processed. Monitoring these orders is essential to avoid delays in order fulfillment and improve operational efficiency.

Problem

The organization lacked a centralized view to monitor orders that were held in the credit pool. Analysts needed to manually review multiple systems to understand the status of each order and the reasons behind the credit hold.

Solution

A monitoring dashboard was developed using Power BI. The data is extracted daily from SQL Server using a custom query and stored in a table to track historical behavior.

  • Orders currently in the credit pool
  • Reasons for credit hold
  • Processing time in the pool
  • Operational monitoring metrics

Impact

The dashboard provides a clear operational overview of orders in the pool, allowing teams to quickly identify bottlenecks and prioritize orders that require attention.
