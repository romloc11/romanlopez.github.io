SQL Data Warehouse Project

Problematic

Before this project, there was no centralized data repository. Analysts had to extract information manually from multiple systems, clean it, merge it, and then perform their analysis. This process was time-consuming and prevented automated reporting.

Solution

I proposed creating a Data Warehouse (DWH) with the necessary information, centralized and cleaned, to enable efficient analysis for the Accounts Receivable area.

  • Designed the architecture with Bronze, Silver, and Gold layers.
  • Implemented a Star Schema for the dimensional model.
  • Documented all tables and processes in the DWH.

Benefits

  • Analysts can query the DWH directly, saving time and avoiding manual work.
  • Golden Layer can be connected to Power BI following the Star Schema.
  • All tables and data are documented.
  • Data is clean and consistent across the organization.
  • The DWH updates automatically, enabling automated reporting.
